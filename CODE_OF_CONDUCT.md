# Code of Conduct

## Our Pledge

We want VoiceVibe to be a project where contributors, users, and maintainers can collaborate without harassment or exclusion.

Participation in this project means helping build a respectful, technically rigorous, and practical community for everyone, regardless of age, body size, disability, ethnicity, gender identity or expression, experience level, education, nationality, personal appearance, race, religion, or sexual identity and orientation.

## Our Standards

Examples of behavior that helps this community:

- Give technical feedback that is specific, respectful, and actionable.
- Assume good intent, but stay precise about problems and tradeoffs.
- Welcome contributors with different levels of open-source experience.
- Keep discussions focused on the product, code, users, and evidence.
- Respect maintainers' scope decisions, especially the desktop-first direction of the project.

Examples of unacceptable behavior:

- Personal attacks, insults, or derogatory comments.
- Harassment, intimidation, threats, or sustained disruption.
- Publishing private information without explicit permission.
- Bad-faith arguments, trolling, or repetitive hostile behavior.
- Sexualized language or imagery in project spaces.
- Pressuring maintainers to accept work outside the project's stated scope.

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. They may remove, edit, or reject comments, commits, code, issues, and other contributions that violate these rules.

## Scope

This Code of Conduct applies in all project spaces, including:

- GitHub issues
- pull requests
- code review discussions
- release discussions
- linked community spaces used for project collaboration

It also applies when someone officially represents the project in public.

## Reporting

If you experience or witness unacceptable behavior, report it privately to the maintainers through GitHub:

- https://github.com/yeyitech

If the report is about repository security, use the process in `SECURITY.md` instead of filing a public issue.

Maintainers will review reports promptly and handle them as confidentially as practical.

## Enforcement Guidelines

Maintainers may choose any response they believe is appropriate, including:

- warning the person involved
- removing content
- temporarily limiting participation
- permanently banning a contributor from project spaces
